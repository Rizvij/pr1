## Adding New Documentation

### For Foundational Documentation
1. Add to appropriate directory (`architecture/`, `development/`, etc.)
2. Keep documentation updated as the system evolves
3. Use descriptive filenames in kebab-case

### For New Features
1. Create folder: `features/YYYY-MM-feature-name/`
2. Add implementation guides and deployment steps
3. Include dates in document headers for reference
